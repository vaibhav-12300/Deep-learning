Lec 56 — Image Denoising & Segmentation Loss
01
Dice Loss
Image Segmentation Loss Function
Lec 55–56
⭐ HIGH PROB MCQ
🇮🇳 मराठी स्पष्टीकरण
Dice Loss हे Image Segmentation साठी वापरले जाणारे एक loss function आहे. हे दोन regions (predicted आणि actual) किती overlap होतात त्यावर आधारित आहे. जर दोन्ही पूर्णपणे match झाल्या → Dice = 1 (perfect). अजिबात match नाही → Dice = 0.

याचा उपयोग तेव्हा होतो जेव्हा class imbalance असतो — उदा. medical images मध्ये tumor खूप लहान असतो आणि background खूप मोठा.
🧩 समजा दोन puzzle pieces आहेत — Dice मोजतो ते किती overlap करतात.
📝 Exam Answer — हे लिहा
  • Dice Loss is based on the Dice coefficient — a measure of overlap between two samples (predicted mask and target mask).
  • Range: 0 to 1 where 1 = perfect overlap.
  • Formula: Dice = 2|A∩B| / (|A| + |B|)
  • |A∩B| = element-wise multiplication of prediction and target, then summed.
  • Dice Loss = 1 − Dice Coefficient
  • More robust than cross-entropy for class-imbalanced segmentation tasks (e.g., small tumors in medical imaging).
  • Multi-class Dice Loss: compute Dice per class, then average.
⚡ Quick Trick / Memory Aid
D = 2 × OVERLAP / TOTAL "2 times intersection over sum of both sets"

का "2"? → कारण intersection A आणि B दोन्हींमध्ये count होते, म्हणून दुप्पट करतो.

Dice = 1 → Perfect Dice = 0 → No overlap Loss = 1 − Dice
Lec 57–59 — Variational Autoencoder
02
Variational Autoencoder (VAE)
Generative Model — Traditional AE च्या मर्यादा + VAE Solution
Lec 57–59
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Traditional Autoencoder चे problem: हे input ला एक fixed point (deterministic code) assign करते. Latent space discontinuous असतो — म्हणजे clusters मध्ये gap असतो. अशा unknown points मधून decode केले तर output garbage येतो.

VAE solution: प्रत्येक input साठी एक point नाही, तर एक distribution (mean μ आणि variance σ) शिकते. Code या distribution मधून sample केला जातो. यामुळे latent space smooth आणि continuous होतो — नवीन images generate करता येतात!
☁️ AE = GPS pin (एक exact point). VAE = GPS "circle" (एक range जिथे तुम्ही असू शकता).
📝 Exam Answer
  • Traditional AE problem: Creates discontinuous latent space → decoding from unknown regions gives unrealistic output.
  • VAE: Encoder outputs μ (mean vector) and σ (std deviation) for each latent dimension.
  • Latent code z is sampled from distribution N(μ, σ²) instead of being a fixed point.
  • Prior on latent space: P(z) = N(0, I) (standard normal).
  • VAE Total Loss = Reconstruction Loss + KL Divergence Loss
  • Reconstruction loss → class-specific structure (clusters).
  • KL Divergence loss → forces encodings toward N(0,I), smooth latent space.
  • Only KL loss → random packing with no structure. Only reconstruction → discontinuous clusters. Both together → ideal smooth latent space.
⚡ Quick Trick
AE = Fixed Point 📍 | VAE = Cloud ☁️ Encoder → μ, σ Sample z ~ N(μ, σ²) Loss = Recon + KL

Vector Arithmetic: "face with glasses" code − "face without glasses" code = "glasses code". फक्त continuous latent space मध्ये शक्य!
03
KL Divergence
VAE Loss Component — Distribution Matching
Lec 58–59
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
KL Divergence म्हणजे दोन probability distributions किती वेगळ्या आहेत ते मोजणारे measure आहे.

KL = 0 → दोन्ही distributions एकच आहेत.
KL जास्त → खूप वेगळ्या आहेत.

VAE मध्ये: encoder च्या distribution Q(z|x) ला prior P(z) = N(0,I) च्या जवळ ठेवण्यासाठी KL loss वापरतो. याला Variational Inference म्हणतात.
📏 KL = "दोन distributions मधील अंतर" — शून्य म्हणजे एकच, जास्त म्हणजे खूप वेगळे.
📝 Exam Answer
  • KL Divergence measures how much distribution Q diverges from distribution P.
  • Formula: KL(P||Q) = Σ P(x) log[P(x)/Q(x)]
  • Asymmetric: KL(P||Q) ≠ KL(Q||P)
  • KL = 0 only when P = Q.
  • In VAE: minimize KL(Q(z|x) || P(z)) where P(z) = N(0, I)
  • Closed-form KL between two Gaussians: KL = 0.5 × [tr(Σ) + μᵀμ − k − log det(Σ)]
  • Variational Lower Bound (ELBO) = E[log P(x|z)] − KL(Q(z|x)||P(z)) — this is what we maximize.
⚡ Quick Trick
"दोन distributions मधील अंतर" KL = 0 → Same distributions KL ≥ 0 always NOT symmetric!

VAE objective: Maximize ELBO = Maximize reconstruction + Minimize KL
Remember: "Maximize likelihood, minimize divergence"
04
Reparameterization Trick
VAE Training को Backprop-Friendly बनवणे
Lec 59
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Problem: VAE मध्ये z = distribution मधून sample करावे लागते. पण sampling हे random operation आहे — त्यातून gradient pass होत नाही. Backpropagation तुटते!

Solution (Reparameterization Trick): Randomness वेगळी काढा. ε (epsilon) ला N(0,I) मधून sample करा, आणि: z = μ + σ × ε

आता μ आणि σ learnable parameters आहेत, आणि gradient त्यांच्यातून जाऊ शकतो!
🎲 "Randomness बाहेर काढा" — ε external noise आहे (network बाहेर), पण μ आणि σ network शिकते.
📝 Exam Answer
  • Problem: Direct sampling from Q(z|x) is a non-differentiable operation → breaks backpropagation (no gradient flow).
  • Solution: Separate randomness from learnable parameters.
  • Sample ε ~ N(0, I) externally, then compute: z = μ + σ ⊙ ε
  • ⊙ = element-wise multiplication.
  • Now gradients can flow through μ and σ (both trainable).
  • ε is an external random variable — not part of the graph.
  • Enables optimization of distribution parameters while preserving the ability to sample randomly.
⚡ Quick Trick
z = μ + σ × ε  |  ε ~ N(0,I) बाहेरून Problem: Sampling breaks gradient Fix: ε = external noise z = μ + σε (deterministic graph)

Think: "Shift a standard Gaussian ε by μ and scale by σ" — same result but now differentiable!
Lec 60 — Generative Adversarial Network
05
GAN — Generative Adversarial Network
Generator vs Discriminator — Game Theory Approach
Lec 60
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
GAN मध्ये दोन networks एकमेकांशी स्पर्धा करतात:

🎭 Generator (G): random noise z घेतो आणि realistic fake images बनवतो. Discriminator ला फसवायचे असते.
🕵️ Discriminator (D): real vs fake image ओळखतो. D(x) = probability of being real.

Training: D आणि G alternately update होतात. G जेव्हा D ला पूर्णपणे फसवू शकतो तेव्हा training converge होते.
🖨️ G = Counterfeiter (बनावट नोटा बनवणारा), D = Detective (बनावट पकडणारा) — दोन्ही एकमेकांना improve करतात.
📝 Exam Answer
  • GAN = Game Theoretic (Minimax) Approach — two networks competing.
  • Generator G: Samples z from Gaussian/Uniform → maps to data point x̃. Tries to fool D.
  • Discriminator D: D(x) = probability x is real. Tries to distinguish real from fake.
  • D Objective (maximize): E[log D(x)] + E[log(1 − D(G(z)))]
  • G Objective (minimize D's / maximize): E[log D(G(z))]
  • Training alternates: k steps D update, then 1 step G update.
  • GAN Applications: High-res image synthesis, image-to-image translation (pix2pix), video-to-video synthesis, image inpainting.
  • VAE vs GAN: VAE explicit likelihood + continuous latent; GAN implicit distribution, sharper images.
⚡ Quick Trick
🎭 G = Cheat | D = Cop | They Train Each Other G: maximize D(G(z)) D: maximize D(real) + minimize D(fake) z ~ Gaussian → G → x̃

Implicit generative model — no need to explicitly define P(x). Generator just maps z to data space.
Lec 51 — Face Recognition
06
Triplet Loss & FaceNet
One-Shot Learning — Face Recognition Embedding
Lec 51
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
One-Shot Learning: खूप कमी (1 किंवा काही) training samples वरून object ओळखणे. Face recognition साठी हे critical आहे — कंपनीत प्रत्येक व्यक्तीचा फक्त 1 फोटो असतो.

Triplet Loss: तीन images वापरतो:
🔵 Anchor (A) — reference face
✅ Positive (P) — same person (A सारखाच)
❌ Negative (N) — different person

Network शिकते: A-P distance कमी, A-N distance जास्त ठेवायला. Margin α हे guarantee करते की gap नेहमी असेल.
🧲 Same person = Magnet attracts | Different person = Magnet repels
📝 Exam Answer
  • One-Shot Learning: Learning from 1 or few training samples per class.
  • FaceNet: Uses deep CNN + L2 normalization to learn embedding function f(x).
  • Triplet = (Anchor A, Positive P, Negative N)
  • Constraint: ||f(A)−f(P)||² + α ≤ ||f(A)−f(N)||²
  • Margin α prevents trivial solution where all embeddings collapse to zero.
  • Triplet Loss: L = max(||f(A)−f(P)||² − ||f(A)−f(N)||² + α, 0)
  • Hard positive: argmax d(A,P) — most dissimilar same-person image.
  • Hard negative: argmin d(A,N) — most similar different-person image.
  • Semi-hard negative preferred → avoids model collapse (f(x)=0).
  • Face Verification: d(reference, query) = ||f(ref) − f(query)||²
⚡ Quick Trick
A-P CLOSE 🟢 | A-N FAR 🔴 | Margin α keeps the gap Triplet = Anchor, Positive, Negative d(A,P) < d(A,N) − α Semi-hard > Hard negative (safer)

Why NOT just minimize d(A,P) = 0? → Because then d(A,N) = 0 too (trivial). Margin α forces separation.
Lec 53–55 — Semantic Segmentation
07
Semantic vs Instance Segmentation + FCN
Pixel-Level Classification
Lec 53–55
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Semantic Segmentation: प्रत्येक pixel ला class label द्या. 3 cars असतील तर तिन्ही "car" च्या label खाली. Image classification at pixel level.

Instance Segmentation: प्रत्येक object ला स्वतंत्रपणे ओळखा. Car-1, Car-2, Car-3 — वेगवेगळ्या instances. Object boundaries at pixel level.

FCN (Fully Convolutional Network): FC layers ऐवजी Convolution layers वापरतो, जेणेकरून pixel-wise output मिळेल.
🏙️ Semantic = "City मध्ये cars आहेत." Instance = "Car #1 इथे, Car #2 तिथे."
📝 Exam Answer
FeatureSemantic SegmentationInstance Segmentation
OutputClass label per pixelIndividual object boundaries
Multiple same-class objectsSame label for allDifferent label for each
Detail levelClass-levelInstance-level

  • FCN: Replaces fully connected layers with conv layers → any input size → pixel-wise output.
  • FCN-32s: 32× upsampling from conv7 → output is coarse/sparse.
  • FCN-16s/8s: Skip connections from earlier layers → finer output.
  • Deconvolution = learnable upsampling. Deconv layers are "reverse of conv."
  • Pixel-wise Cross Entropy: L = −(1/N) Σ p(x,y)·log q(x,y)
  • Applications: Autonomous driving, medical imaging (brain MRI segmentation).
⚡ Quick Trick
Semantic = Crowd | Instance = Individual 🙋 FCN = No FC layers! FCN-32s → coarse (less detail) Skip connections → finer output

FCN-32s, FCN-16s, FCN-8s — number = upsampling factor. Smaller number = better detail.
Lec 46–49 — Normalization
08
Batch / Layer / Instance / Group Normalization
Internal Covariate Shift — कोणत्या Dimension वर Normalize?
Lec 46–49
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Covariate Shift: training दरम्यान प्रत्येक layer च्या input distribution बदलते → training slow होते. Normalization हे fix करते.

4 types — फरक फक्त कोणत्या dimension वर normalize करतो यात आहे:
📦 Batch Norm: Batch dimension (N) वर — प्रत्येक channel साठी.
📋 Layer Norm: प्रत्येक sample साठी सर्व channels वर — NLP/Transformers.
🎨 Instance Norm: प्रत्येक sample, प्रत्येक channel च्या spatial dimensions वर — Style Transfer.
👥 Group Norm: Channels च्या groups वर — Small batch साठी उत्तम.
📊 सर्व normalize करतात, पण "कोणाच्या perspective ने" हे वेगळे आहे.
📝 Exam Answer
TypeNormalizes OverBest For
Batch NormN (batch) per channelLarge batch CNN
Layer NormAll C, H, W per sampleRNN, Transformers
Instance NormH × W per sample per channelStyle Transfer
Group NormGroups of channels per sampleSmall batch size

  • Batch Norm addresses Internal Covariate Shift (Ioffe & Szegedy, 2015).
  • BN allows higher learning rates + acts as regularizer.
  • BN formula: ŷ = γ·(x−μ)/σ + β (γ, β are learnable).
  • Group Norm (Kaiming He, 2018) — better than BN for small batches.
⚡ Quick Trick
B-L-I-G → "Big Lions In Groups" 🦁 B = across Batch (N) L = across all Layers/Channels per sample I = per channel H×W (Instance) G = Groups of channels

Small batch? → Group Norm. Large batch? → Batch Norm. Transformer? → Layer Norm.
Lec 50 — Training Tricks
09
Dropout, Regularization & Early Stopping
Overfitting रोखण्याचे उपाय
Lec 50
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Overfitting: Model training data वर खूप चांगला perform करतो, पण test data वर खराब — noise शिकतो.
Underfitting: Model पुरेसा शिकत नाही — simple model, complex problem.

Dropout: Training मध्ये randomly neurons बंद करतो (probability p). Network कोणत्याही एकाच neuron वर अवलंबून राहत नाही. Testing मध्ये सर्व neurons ON असतात.

Early Stopping: Validation loss बघत train करा, जेव्हा validation loss वाढू लागते तेव्हा थांबा.
🎲 Dropout = "कधी कोणाला बोलावा हे माहीत नाही" → सर्वजण तयार राहतात.
📝 Exam Answer
  • Overfitting: Model fits noise in training data. High train accuracy, low test accuracy.
  • Regularization = Loss + Regularization Term
  • L1 Reg: Adds |w| — encourages sparsity (some weights → 0).
  • L2 Reg (Weight Decay): Adds w² — prefers smaller weights.
  • Dropout (Srivastava et al.): Randomly drops neurons during training with probability p.
  • Training: neurons dropped. Testing: all neurons active (weights scaled by (1−p)).
  • Effect: Dropout learns more robust features (visible in MNIST experiment).
  • Data Augmentation: Rotation, flipping, scaling, contrast enhancement — increases training set size.
  • Early Stopping: Train → evaluate on validation each epoch → save best model → stop when val loss stops improving → use best saved model for testing.
⚡ Quick Trick
🎲 TRAIN = Random OFF | TEST = ALL ON L1 → Sparse (some zero) L2 → Small (all smaller) Dropout → p = drop probability Early stop → Val loss = monitor

Cost = Loss + λ||w||₁ (L1)  or  Cost = Loss + λ||w||₂² (L2)
Lec 43–45 — Gradient Descent Optimization
10
Optimizers: Momentum → Adagrad → RMSProp → Adam
SGD च्या पुढे — Adaptive Learning Rate Methods
Lec 43–45
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
SGD (plain) slow असतो. Problems: learning rate manually set करावी लागते, saddle points मध्ये अडकतो.

🔵 Momentum: मागचा velocity लक्षात ठेवतो — ball rolling downhill.
🟢 NAG: Future position ला gradient बघतो (lookahead).
🟡 Adagrad: प्रत्येक parameter साठी वेगळी learning rate — sparse features साठी चांगला. Problem: LR खूप लवकर कमी होतो.
🟠 RMSProp: Adagrad चे fix — exponentially decaying average वापरतो, जुनी history विसरतो.
🔴 Adam: Momentum + RMSProp = सर्वात popular optimizer!
🚀 SGD = Walking | Momentum = Running | Adam = Car with GPS!
📝 Exam Answer
OptimizerKey IdeaWeakness
SGDBasic gradient updateSlow, manual LR, saddle points
MomentumAccumulates velocity (past gradients)Manual hyperparameters
NAGGradient at lookahead positionManual hyperparameters
AdagradAdaptive LR per param (÷ √Σg²)LR → 0 for non-convex
RMSPropExponentially decaying avg of g²—
AdamMomentum + RMSProp + bias correctionBest general choice ✓

  • Adam update: ŝ = s/(1−β₁), r̂ = r/(1−β₂), W = W − η·ŝ/√r̂
  • Typical values: β₁ = 0.9, β₂ = 0.999
  • Adagrad problem: accumulation of all historical gradients → LR → 0 (bad for non-convex).
  • Saddle points: gradient ≈ 0 in all dims → SGD gets stuck. Momentum escapes saddle points.
⚡ Quick Trick
SGD → Momentum → Adagrad → RMSProp → Adam 🏆 Adam = Momentum + RMSProp Adagrad problem = LR dies RMSProp = Adagrad + forgetting β₁=0.9, β₂=0.999 for Adam

Adagrad: "Too honest — remembers every mistake forever"
RMSProp: "Forgives old mistakes, focuses on recent"

ResNet — Residual Network & Skip Connections
Vanishing Gradient Problem चे Solution
Lec 42
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Deep network मध्ये जितके जास्त layers, तितका vanishing gradient problem. ResNet ने एक simple trick केली — Skip Connection (Identity Shortcut).

Layer F(x) शिकण्याऐवजी Residual F(x) = H(x) − x शिकते. म्हणजे layer ला फक्त "input पेक्षा किती वेगळे आहे" ते शिकायचे आहे.

Skip connections gradient ला directly earlier layers कडे पाठवतात → vanishing gradient सुटते!
🛤️ Skip connection = Highway — gradient ला लांब जायची "highway" मिळते.
📝 Exam Answer
  • ResNet core idea: Skip/Identity Shortcut Connections that skip one or more layers.
  • Layer learns residual mapping F(x) = H(x) − x, where H(x) = desired mapping.
  • Forward: a_l = f(Z_l + a_{l-2}) (if same dimension)
  • Stacking identity mappings ensures deeper network ≥ shallow network performance.
  • Gradient can flow to any earlier layer through shortcut → alleviates vanishing gradient.
  • Deeper network should NOT give higher training error than shallow network (ResNet solves this).
  • Vanishing gradient solutions: ReLU activation, proper weight initialization, skip connections.
⚡ Quick Trick
H(x) = F(x) + x  →  Learn RESIDUAL, not the full mapping Skip connection = Gradient highway F(x) = H(x) − x (residual) Deeper ≥ Shallower (guaranteed)

Why "residual"? If identity mapping is optimal, it's easier to push F(x) → 0 than to learn H(x) = x directly.

GoogLeNet — Inception Module & Auxiliary Classifiers
ILSVRC 2014 Winner — Multi-scale Feature Extraction
Lec 41–42
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Inception Module: एकाच वेळी 1×1, 3×3, 5×5 convolutions parallel मध्ये apply करतो, आणि outputs concatenate करतो. यामुळे multi-scale features मिळतात — लहान detail (1×1) आणि मोठा context (5×5) दोन्ही.

Auxiliary Classifiers: Deep network मध्ये gradient vanish होतो. Middle layers वर छोटे classifiers लावतात → gradient signal middle layers पर्यंत पोहोचतो. Testing वेळी हे discard होतात.
🔭 Inception = एकाच वेळी different zoom levels ने बघणे (1×1 = zoom in, 5×5 = zoom out).
📝 Exam Answer
  • GoogLeNet (Inception v1): 22 layers with parameters; 27 layers including MaxPool; 9 Inception modules.
  • Inception Module: Applies 1×1, 3×3, 5×5 convolutions and MaxPool in parallel. Outputs stacked along channel dimension. Multi-level feature extractor.
  • 1×1 convolutions also perform dimensionality reduction (reduce computation).
  • Top-5 error rate: < 7%.
  • Auxiliary Classifiers: Smaller CNNs placed on middle Inception modules. Added to handle vanishing gradient in deep network.
  • Auxiliary loss weighted by 0.3 during training. Discarded at inference time.
⚡ Quick Trick
🔭 Inception = 1×1 + 3×3 + 5×5 PARALLEL → Concatenate Aux classifier weight = 0.3 Aux classifiers → discarded at test time 1×1 conv = reduce computation

1×1 conv trick: 480 channels → reduce → then 5×5. Computation: 5.3M vs 112.9M without 1×1. 21× cheaper!
Lec 56 — Image Restoration
13
Image Restoration Network & Skip Connections
Encoder-Decoder + Skip Connections for Denoising
Lec 56
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Image Restoration = खराब (noisy/blurry) image ला clean image मध्ये convert करणे. Network: Encoder (conv) + Decoder (deconv) + Skip Connections.

Skip Connections चा उपयोग: Deep network मध्ये image details हरवतात. Skip connections encoder च्या feature maps directly decoder ला देतात — details preserve होतात. Plus gradient backpropagation सुधारतो.

Application: Low Dose CT Denoising — radiation कमी करण्यासाठी low dose images घेतल्या जातात; Deep Learning network त्या denoise करतो.
📸 Encoder = "Detail काढा" | Decoder = "पुन्हा बनवा" | Skip = "हे details विसरू नका!"
📝 Exam Answer
  • Image Degradation model: Degraded = H(Clean) + Noise (H = degradation operator).
  • If H = Identity matrix → Image denoising.
  • Types of restoration: deblurring, denoising, super resolution, inpainting.
  • Restoration Network: Symmetric Encoder (conv, downsampling) + Decoder (deconv, upsampling) + Skip Connections.
  • Conv layers = feature extractor (eliminate noise). Deconv layers = recover image details.
  • Why Skip Connections?
    — Deep network loses image details as depth increases.
    — Skip connections carry feature maps from encoder directly to decoder → preserve details.
    — Also help backpropagate gradients to bottom layers → easier training.
  • Training loss: L(Θ) = (1/N) Σ ||F(Xᵢ; Θ) − Yᵢ||² (MSE between restored and clean)
  • Network learns additive corruption from input → converges faster than direct mapping.

01
Dice Loss
Image Segmentation Loss Function
Lec 55–56
⭐ HIGH PROB MCQ
🇮🇳 मराठी स्पष्टीकरण
Dice Loss हे Image Segmentation साठी वापरले जाणारे एक loss function आहे. हे दोन regions (predicted आणि actual) किती overlap होतात त्यावर आधारित आहे. जर दोन्ही पूर्णपणे match झाल्या → Dice = 1 (perfect). अजिबात match नाही → Dice = 0.

याचा उपयोग तेव्हा होतो जेव्हा class imbalance असतो — उदा. medical images मध्ये tumor खूप लहान असतो आणि background खूप मोठा.
🧩 समजा दोन puzzle pieces आहेत — Dice मोजतो ते किती overlap करतात.
📝 Exam Answer — हे लिहा
  • Dice Loss is based on the Dice coefficient — a measure of overlap between two samples (predicted mask and target mask).
  • Range: 0 to 1 where 1 = perfect overlap.
  • Formula: Dice = 2|A∩B| / (|A| + |B|)
  • |A∩B| = element-wise multiplication of prediction and target, then summed.
  • Dice Loss = 1 − Dice Coefficient
  • More robust than cross-entropy for class-imbalanced segmentation tasks (e.g., small tumors in medical imaging).
  • Multi-class Dice Loss: compute Dice per class, then average.
⚡ Quick Trick / Memory Aid
D = 2 × OVERLAP / TOTAL "2 times intersection over sum of both sets"

का "2"? → कारण intersection A आणि B दोन्हींमध्ये count होते, म्हणून दुप्पट करतो.

Dice = 1 → Perfect Dice = 0 → No overlap Loss = 1 − Dice
Lec 57–59 — Variational Autoencoder
02
Variational Autoencoder (VAE)
Generative Model — Traditional AE च्या मर्यादा + VAE Solution
Lec 57–59
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Traditional Autoencoder चे problem: हे input ला एक fixed point (deterministic code) assign करते. Latent space discontinuous असतो — म्हणजे clusters मध्ये gap असतो. अशा unknown points मधून decode केले तर output garbage येतो.

VAE solution: प्रत्येक input साठी एक point नाही, तर एक distribution (mean μ आणि variance σ) शिकते. Code या distribution मधून sample केला जातो. यामुळे latent space smooth आणि continuous होतो — नवीन images generate करता येतात!
☁️ AE = GPS pin (एक exact point). VAE = GPS "circle" (एक range जिथे तुम्ही असू शकता).
📝 Exam Answer
  • Traditional AE problem: Creates discontinuous latent space → decoding from unknown regions gives unrealistic output.
  • VAE: Encoder outputs μ (mean vector) and σ (std deviation) for each latent dimension.
  • Latent code z is sampled from distribution N(μ, σ²) instead of being a fixed point.
  • Prior on latent space: P(z) = N(0, I) (standard normal).
  • VAE Total Loss = Reconstruction Loss + KL Divergence Loss
  • Reconstruction loss → class-specific structure (clusters).
  • KL Divergence loss → forces encodings toward N(0,I), smooth latent space.
  • Only KL loss → random packing with no structure. Only reconstruction → discontinuous clusters. Both together → ideal smooth latent space.
⚡ Quick Trick
AE = Fixed Point 📍 | VAE = Cloud ☁️ Encoder → μ, σ Sample z ~ N(μ, σ²) Loss = Recon + KL

Vector Arithmetic: "face with glasses" code − "face without glasses" code = "glasses code". फक्त continuous latent space मध्ये शक्य!
03
KL Divergence
VAE Loss Component — Distribution Matching
Lec 58–59
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
KL Divergence म्हणजे दोन probability distributions किती वेगळ्या आहेत ते मोजणारे measure आहे.

KL = 0 → दोन्ही distributions एकच आहेत.
KL जास्त → खूप वेगळ्या आहेत.

VAE मध्ये: encoder च्या distribution Q(z|x) ला prior P(z) = N(0,I) च्या जवळ ठेवण्यासाठी KL loss वापरतो. याला Variational Inference म्हणतात.
📏 KL = "दोन distributions मधील अंतर" — शून्य म्हणजे एकच, जास्त म्हणजे खूप वेगळे.
📝 Exam Answer
  • KL Divergence measures how much distribution Q diverges from distribution P.
  • Formula: KL(P||Q) = Σ P(x) log[P(x)/Q(x)]
  • Asymmetric: KL(P||Q) ≠ KL(Q||P)
  • KL = 0 only when P = Q.
  • In VAE: minimize KL(Q(z|x) || P(z)) where P(z) = N(0, I)
  • Closed-form KL between two Gaussians: KL = 0.5 × [tr(Σ) + μᵀμ − k − log det(Σ)]
  • Variational Lower Bound (ELBO) = E[log P(x|z)] − KL(Q(z|x)||P(z)) — this is what we maximize.
⚡ Quick Trick
"दोन distributions मधील अंतर" KL = 0 → Same distributions KL ≥ 0 always NOT symmetric!

VAE objective: Maximize ELBO = Maximize reconstruction + Minimize KL
Remember: "Maximize likelihood, minimize divergence"
04
Reparameterization Trick
VAE Training को Backprop-Friendly बनवणे
Lec 59
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Problem: VAE मध्ये z = distribution मधून sample करावे लागते. पण sampling हे random operation आहे — त्यातून gradient pass होत नाही. Backpropagation तुटते!

Solution (Reparameterization Trick): Randomness वेगळी काढा. ε (epsilon) ला N(0,I) मधून sample करा, आणि: z = μ + σ × ε

आता μ आणि σ learnable parameters आहेत, आणि gradient त्यांच्यातून जाऊ शकतो!
🎲 "Randomness बाहेर काढा" — ε external noise आहे (network बाहेर), पण μ आणि σ network शिकते.
📝 Exam Answer
  • Problem: Direct sampling from Q(z|x) is a non-differentiable operation → breaks backpropagation (no gradient flow).
  • Solution: Separate randomness from learnable parameters.
  • Sample ε ~ N(0, I) externally, then compute: z = μ + σ ⊙ ε
  • ⊙ = element-wise multiplication.
  • Now gradients can flow through μ and σ (both trainable).
  • ε is an external random variable — not part of the graph.
  • Enables optimization of distribution parameters while preserving the ability to sample randomly.
⚡ Quick Trick
z = μ + σ × ε  |  ε ~ N(0,I) बाहेरून Problem: Sampling breaks gradient Fix: ε = external noise z = μ + σε (deterministic graph)

Think: "Shift a standard Gaussian ε by μ and scale by σ" — same result but now differentiable!
Lec 60 — Generative Adversarial Network
05
GAN — Generative Adversarial Network
Generator vs Discriminator — Game Theory Approach
Lec 60
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
GAN मध्ये दोन networks एकमेकांशी स्पर्धा करतात:

🎭 Generator (G): random noise z घेतो आणि realistic fake images बनवतो. Discriminator ला फसवायचे असते.
🕵️ Discriminator (D): real vs fake image ओळखतो. D(x) = probability of being real.

Training: D आणि G alternately update होतात. G जेव्हा D ला पूर्णपणे फसवू शकतो तेव्हा training converge होते.
🖨️ G = Counterfeiter (बनावट नोटा बनवणारा), D = Detective (बनावट पकडणारा) — दोन्ही एकमेकांना improve करतात.
📝 Exam Answer
  • GAN = Game Theoretic (Minimax) Approach — two networks competing.
  • Generator G: Samples z from Gaussian/Uniform → maps to data point x̃. Tries to fool D.
  • Discriminator D: D(x) = probability x is real. Tries to distinguish real from fake.
  • D Objective (maximize): E[log D(x)] + E[log(1 − D(G(z)))]
  • G Objective (minimize D's / maximize): E[log D(G(z))]
  • Training alternates: k steps D update, then 1 step G update.
  • GAN Applications: High-res image synthesis, image-to-image translation (pix2pix), video-to-video synthesis, image inpainting.
  • VAE vs GAN: VAE explicit likelihood + continuous latent; GAN implicit distribution, sharper images.
⚡ Quick Trick
🎭 G = Cheat | D = Cop | They Train Each Other G: maximize D(G(z)) D: maximize D(real) + minimize D(fake) z ~ Gaussian → G → x̃

Implicit generative model — no need to explicitly define P(x). Generator just maps z to data space.
Lec 51 — Face Recognition
06
Triplet Loss & FaceNet
One-Shot Learning — Face Recognition Embedding
Lec 51
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
One-Shot Learning: खूप कमी (1 किंवा काही) training samples वरून object ओळखणे. Face recognition साठी हे critical आहे — कंपनीत प्रत्येक व्यक्तीचा फक्त 1 फोटो असतो.

Triplet Loss: तीन images वापरतो:
🔵 Anchor (A) — reference face
✅ Positive (P) — same person (A सारखाच)
❌ Negative (N) — different person

Network शिकते: A-P distance कमी, A-N distance जास्त ठेवायला. Margin α हे guarantee करते की gap नेहमी असेल.
🧲 Same person = Magnet attracts | Different person = Magnet repels
📝 Exam Answer
  • One-Shot Learning: Learning from 1 or few training samples per class.
  • FaceNet: Uses deep CNN + L2 normalization to learn embedding function f(x).
  • Triplet = (Anchor A, Positive P, Negative N)
  • Constraint: ||f(A)−f(P)||² + α ≤ ||f(A)−f(N)||²
  • Margin α prevents trivial solution where all embeddings collapse to zero.
  • Triplet Loss: L = max(||f(A)−f(P)||² − ||f(A)−f(N)||² + α, 0)
  • Hard positive: argmax d(A,P) — most dissimilar same-person image.
  • Hard negative: argmin d(A,N) — most similar different-person image.
  • Semi-hard negative preferred → avoids model collapse (f(x)=0).
  • Face Verification: d(reference, query) = ||f(ref) − f(query)||²
⚡ Quick Trick
A-P CLOSE 🟢 | A-N FAR 🔴 | Margin α keeps the gap Triplet = Anchor, Positive, Negative d(A,P) < d(A,N) − α Semi-hard > Hard negative (safer)

Why NOT just minimize d(A,P) = 0? → Because then d(A,N) = 0 too (trivial). Margin α forces separation.

Dropout, Regularization & Early Stopping
Overfitting रोखण्याचे उपाय
Lec 50
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Overfitting: Model training data वर खूप चांगला perform करतो, पण test data वर खराब — noise शिकतो.
Underfitting: Model पुरेसा शिकत नाही — simple model, complex problem.

Dropout: Training मध्ये randomly neurons बंद करतो (probability p). Network कोणत्याही एकाच neuron वर अवलंबून राहत नाही. Testing मध्ये सर्व neurons ON असतात.

Early Stopping: Validation loss बघत train करा, जेव्हा validation loss वाढू लागते तेव्हा थांबा.
🎲 Dropout = "कधी कोणाला बोलावा हे माहीत नाही" → सर्वजण तयार राहतात.
📝 Exam Answer
  • Overfitting: Model fits noise in training data. High train accuracy, low test accuracy.
  • Regularization = Loss + Regularization Term
  • L1 Reg: Adds |w| — encourages sparsity (some weights → 0).
  • L2 Reg (Weight Decay): Adds w² — prefers smaller weights.
  • Dropout (Srivastava et al.): Randomly drops neurons during training with probability p.
  • Training: neurons dropped. Testing: all neurons active (weights scaled by (1−p)).
  • Effect: Dropout learns more robust features (visible in MNIST experiment).
  • Data Augmentation: Rotation, flipping, scaling, contrast enhancement — increases training set size.
  • Early Stopping: Train → evaluate on validation each epoch → save best model → stop when val loss stops improving → use best saved model for testing.
⚡ Quick Trick
🎲 TRAIN = Random OFF | TEST = ALL ON L1 → Sparse (some zero) L2 → Small (all smaller) Dropout → p = drop probability Early stop → Val loss = monitor

Cost = Loss + λ||w||₁ (L1)  or  Cost = Loss + λ||w||₂² (L2)

SGD च्या पुढे — Adaptive Learning Rate Methods
Lec 43–45
⭐⭐ VERY HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
SGD (plain) slow असतो. Problems: learning rate manually set करावी लागते, saddle points मध्ये अडकतो.

🔵 Momentum: मागचा velocity लक्षात ठेवतो — ball rolling downhill.
🟢 NAG: Future position ला gradient बघतो (lookahead).
🟡 Adagrad: प्रत्येक parameter साठी वेगळी learning rate — sparse features साठी चांगला. Problem: LR खूप लवकर कमी होतो.

🚀 SGD = Walking | Momentum = Running | Adam = Car with GPS!

SGDBasic gradient updateSlow, manual LR, saddle points
MomentumAccumulates velocity (past gradients)Manual hyperparameters
NAGGradient at lookahead positionManual hyperparameters

ResNet — Residual Network & Skip Connections
Vanishing Gradient Problem चे Solution
Lec 42
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Deep network मध्ये जितके जास्त layers, तितका vanishing gradient problem. ResNet ने एक simple trick केली — Skip Connection (Identity Shortcut).

Layer F(x) शिकण्याऐवजी Residual F(x) = H(x) − x शिकते. म्हणजे layer ला फक्त "input पेक्षा किती वेगळे आहे" ते शिकायचे आहे.

Skip connections gradient ला directly earlier layers कडे पाठवतात → vanishing gradient सुटते!
🛤️ Skip connection = Highway — gradient ला लांब जायची "highway" मिळते.
📝 Exam Answer
  • ResNet core idea: Skip/Identity Shortcut Connections that skip one or more layers.
  • Layer learns residual mapping F(x) = H(x) − x, where H(x) = desired mapping.
  • Forward: a_l = f(Z_l + a_{l-2}) (if same dimension)
  • Stacking identity mappings ensures deeper network ≥ shallow network performance.
  • Gradient can flow to any earlier layer through shortcut → alleviates vanishing gradient.
  • Deeper network should NOT give higher training error than shallow network (ResNet solves this).
  • Vanishing gradient solutions: ReLU activation, proper weight initialization, skip connections.
⚡ Quick Trick
H(x) = F(x) + x  →  Learn RESIDUAL, not the full mapping Skip connection = Gradient highway F(x) = H(x) − x (residual) Deeper ≥ Shallower (guaranteed)

Why "residual"? If identity mapping is optimal, it's easier to push F(x) → 0 than to learn H(x) = x directly.

Image Restoration Network & Skip Connections
Encoder-Decoder + Skip Connections for Denoising
Lec 56
⭐ HIGH PROB
🇮🇳 मराठी स्पष्टीकरण
Image Restoration = खराब (noisy/blurry) image ला clean image मध्ये convert करणे. Network: Encoder (conv) + Decoder (deconv) + Skip Connections.

Skip Connections चा उपयोग: Deep network मध्ये image details हरवतात. Skip connections encoder च्या feature maps directly decoder ला देतात — details preserve होतात. Plus gradient backpropagation सुधारतो.

Application: Low Dose CT Denoising — radiation कमी करण्यासाठी low dose images घेतल्या जातात; Deep Learning network त्या denoise करतो.
📸 Encoder = "Detail काढा" | Decoder = "पुन्हा बनवा" | Skip = "हे details विसरू नका!"
📝 Exam Answer
  • Image Degradation model: Degraded = H(Clean) + Noise (H = degradation operator).
  • If H = Identity matrix → Image denoising.
  • Types of restoration: deblurring, denoising, super resolution, inpainting.
  • Restoration Network: Symmetric Encoder (conv, downsampling) + Decoder (deconv, upsampling) + Skip Connections.
  • Conv layers = feature extractor (eliminate noise). Deconv layers = recover image details.
  • Why Skip Connections?
    — Deep network loses image details as depth increases.
    — Skip connections carry feature maps from encoder directly to decoder → preserve details.
    — Also help backpropagate gradients to bottom layers → easier training.
  • Training loss: L(Θ) = (1/N) Σ ||F(Xᵢ; Θ) − Yᵢ||² (MSE between restored and clean)
  • Network learns additive corruption from input → converges faster than direct mapping.
⚡ Quick Trick
Encoder ↓ → Bottleneck → Decoder ↑ + Skip connections ↔ H = Identity → Denoising Skip = preserves details + helps gradient Loss = MSE (clean vs restored)

Low Dose CT: current → lower dose (less radiation) → noisy image → DL network denoise → clean CT. Same as image denoising problem!
